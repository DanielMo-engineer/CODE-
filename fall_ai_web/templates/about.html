📌 项目概况

AI 长者看护系统

基于 D-Robotics RDK X5 边缘计算平台的智能看护系统，面向独居老人的安全监护需求。系统利用 BPU 加速的 YOLO11n-pose 模型实现实时人体姿态估计与摔倒检测，结合表情识别、AI 对话等功能，提供全方位的智能看护解决方案。

🖥️ 硬件平台

RDK X5 规格

  • SoC: D-Robotics RDK X5 (8 核 ARM Cortex-A)
  • BPU: 双核神经网络处理器，支持 INT8 量化推理
  • 内存: 8GB LPDDR4
  • 存储: 64GB eMMC (实际可用约 -- GB)
  • AI 算力: 约 10 TOPS (INT8)
  • 摄像头: USB 摄像头 / MIPI CSI 接口
  • 网络: 千兆以太网 + WiFi 6 + Bluetooth 5

🔧 关键技术决策

1️⃣ 单模型姿态 + 摔倒

使用 YOLO11n-pose 单模型同时输出人体检测框 + 17 个 COCO 关键点，在 BPU 上 INT8 量化推理。相比"检测 + 姿态双模型"方案：

  • ✅ 推理延迟降低约 40%
  • ✅ 内存占用减少约 30%
  • ✅ 摔倒检测融合姿态信息（人体倾角 + 水平比例）
  • ✅ 骨架可视化直接来源于关键点数据

2️⃣ 三级过滤摔倒检测

为降低误报率，采用三级过滤策略：

  • 第一级 (框形状): 人体框宽高比 > 1.3 (水平姿态)
  • 第二级 (速度): 垂直移动速度 > 阈值 (快速倒下)
  • 第三级 (姿态): 人体倾角 + 水平度验证
  • 持久化过滤: 连续 N 帧满足条件才触发报警

3️⃣ 无代理直连视频流

摄像头服务器（8080 端口）的 HTTPerver 是单线程模型。Flask 代理 MJPEG 流会阻塞其他 API 请求。最终方案：浏览器直连摄像头服务器，Flask 仅提供 socket 代理作为备选。

4️⃣ 边缘 + 云端混合架构

  • 边缘 (RDK X5): 实时视频推理、摔倒检测、表情识别 — 延迟 <50ms
  • 云端 (DeepSeek): AI 对话、心情日志总结 — 需要网络，但非实时任务
  • 告警: 本地第一时间触发，Server 酱微信推送作为辅助

5️⃣ PWA + mDNS 网络

  • PWA: Service Worker 缓存应用外壳，离线时显示离线页面
  • mDNS: 通过 rdkx5.local:5050 访问，永不丢 IP
  • cpolar: 公网隧道，外网也能查看（延迟 ~5-6s 免费版）
  • 开机自启: systemd 服务 fall-platform.service
