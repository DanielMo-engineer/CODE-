# RDK X5 人体摔倒检测系统 — 项目总结

## 🎯 项目目标

基于地平线 RDK X5 开发板，利用 BPU（神经网络加速器）的实时 AI 推理能力，实现：
- **实时人体检测 + 姿态估计**
- **摔倒行为自动判别**
- **多渠道告警通知**（微信/蜂鸣器/语音电话）
- **Web 画面推流**，远程查看

## 🔬 摔倒检测算法详解

### 分析维度

| 维度 | 描述 | 典型值 |
|------|------|--------|
| **框宽高比** | 人体框宽度/高度，摔倒后变扁 | 站立 <0.5，摔倒 >1.0 |
| **垂直速度** | 人体中心点在 y 方向的变化速率 | 正常 <0.3 身/s，摔倒 >0.7 身/s |
| **身体倾角** | 肩髋连线与水平面的夹角 | 站立 ≈90°，摔倒 <30° |
| **水平速度** | 横向移动速度，区分主动躺倒 vs 摔倒 | 摔倒时垂直 > 水平 |

### 三级过滤器「三选二」投票

```
Filter 1: 垂直下降为主？      (vert_vel > horiz_vel × 1.5)
Filter 2: 躺平时有位移？      (摔倒会有惯性滑动)
Filter 3: 框面积稳定？        (帧间抖动不超过50%)
→ 通过 ≥2 个才确认是有效摔倒
```

### 抗误判措施

| 措施 | 作用 |
|------|------|
| 🆕 **关键点数量过滤** | 有效节点 ≤10 时不判摔倒，避免远距/遮挡误报 |
| **连续帧确认** | 同一状态需持续 N 帧才切换状态 |
| **告警冷却** | 10 秒内不重复推送 |
| **近距离收紧** | 人离摄像头近时阈值更严格 |
| **姿态仲裁** | 有姿态信息时要求身体确实躺平才触发 |

## 🌐 如何查看实时画面

启动系统后，局域网内任意设备打开浏览器访问：

```
http://设备IP地址:8080
```

画面包含：
- **彩色骨架** — 关键点连线跟随人体动作
- **状态栏** — 当前状态 / 帧率 / 人数
- **指标面板** — 宽高比 / 速度 / 连续帧计数
- **绿色框** = 正常，**红色框** = 摔倒报警
- 摔倒时骨架颜色变暗红

## 📝 参数调优参考

| 参数 | 默认值 | 调大 → | 调小 → |
|------|--------|--------|--------|
| `--score` | 0.25 | 检测更严格，漏检增多 | 检测更灵敏，误报增多 |
| `--kpt-conf` | 0.3 | 关键点更少但更准 | 关键点更多但有杂点 |
| `--confirm` | 10 帧 | 需要更久确认摔倒 | 响应更快但可能误报 |
| `--cooldown` | 5 秒 | 告警间隔更长 | 告警更频繁 |
